# Slide Patterns

Use these as concrete references when extending the current Paper deck.

## Main System

- Backgrounds: slightly off-white, warm, calm.
- Typography: `SF Pro Display`, large black titles, short quiet footer text.
- Composition: one central graphic, no clutter, obvious read path.
- Footer: thin divider line plus one short speaker-cue sentence.

## Archetypes

### Overview Arc

- Example in current deck: `Overview Slide Simple`
- Pattern: one simple progression across the page
- Use when summarizing a journey or chapter map

### Modality Slide

- Example in current deck: `Text to Text`
- Pattern: large title plus one centered prompt-to-output graphic
- Use when teaching one AI capability at a time

### Concept Explainer

- Example in current deck: `APIs`
- Pattern: plain-language definition plus one simple metaphor
- Use when introducing a new idea that needs translation into simple terms

### Vibe Coding Variant

- Examples in current deck: slides `18` through `25`
- Pattern: inset border, top-right chapter label/number, warmer chapter feel
- Use when signaling a section break without abandoning the overall deck language

### Prompt -> Code -> Output

- Example in current deck: `Remotion`
- Pattern: left prompt, middle code, right output
- Use when a slide needs to show creation flow directly

## Practical Rules

- Keep each slide readable in a few seconds.
- If a slide has more than one core graphic, simplify it.
- Prefer deleting elements over shrinking them.
- Let graphics carry the explanation whenever possible.
